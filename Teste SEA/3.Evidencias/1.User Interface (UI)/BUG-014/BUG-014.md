# BUG-014 — Lista "Funcionário(s)": container não comporta aumento de registros (sem expansão/scroll/paginação)

## Tipo
UI / Usabilidade

## Severidade
Média 

## Ambiente
- SO: Windows 11
- Microsoft Edge: Versão 143.0.3650.96 (64 bits)
- Google Chrome: Versão 143.0.7499.170 (64 bits)

## Passos para reproduzir
1. Acessar a aba "Funcionário(s)".
2. Adicionar vários funcionários (até aumentar a quantidade de registros).
3. Observar o comportamento do container/lista na tela.

## Resultado atual (Aplicação)
- O box/container que exibe os funcionários não se ajusta bem ao aumento de registros.
- Conforme a quantidade cresce, a listagem fica limitada visualmente, prejudicando a navegação/visualização completa.

## Resultado esperado (Protótipo)
- A listagem deve se adaptar ao aumento de registros, seguindo uma das abordagens:
  - expansão do container conforme a lista cresce; ou
  - scroll interno com altura definida; ou
  - paginação (limitando itens por página e permitindo navegar entre páginas).

## Impacto
- Prejudica a visualização e navegação da lista conforme aumenta o número de funcionários.
- Pode dificultar acesso às ações dos cards.

## Sugestão de correção
- Definir um comportamento padrão para listagem: scroll interno ou paginação.
- Garantir que o container mantenha boa usabilidade conforme a lista cresce (sem cortar informações e mantendo acessíveis as ações dos cards).
