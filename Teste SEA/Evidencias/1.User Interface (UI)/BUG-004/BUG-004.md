# BUG-004 — Stepper: rótulo "Concluído" não segue padrão do protótipo ao marcar "Etapa concluída = Sim"

## Tipo
UI / Conformidade com Protótipo

## Severidade
Média

## Ambiente
- SO: Windows 10/11
- Microsoft Edge: Versão 143.0.3650.96 (64 bits)
- Google Chrome: Versão 143.0.7499.170 (64 bits)

## Pré-condição
- Definir "A etapa está concluída?" como Sim.

## Passos para reproduzir
1. Acessar a tela com o stepper (itens 1–9).
2. Marcar "A etapa está concluída?" como Sim.
3. Observar o rótulo exibido abaixo do item concluído no stepper.

## Resultado atual (Aplicação)
- O rótulo de conclusão é exibido, porém não segue o padrão do protótipo (ex.: caixa alta, acentuação/capitalização e estilo visual divergentes).

## Resultado esperado (Protótipo)
- Quando "Etapa concluída" = Sim, o item concluído no stepper deve:
  - exibir o rótulo "Concluído" abaixo do item;
  - texto em preto;
  - com acentuação correta e apenas a primeira letra maiúscula;
  - manter o padrão de fonte definido no protótipo (Segoe UI).

## Impacto
- Reduz a clareza sobre o estado da etapa (concluída ou não), pois o indicador visual não segue o padrão esperado.
- Quebra a consistência visual do stepper em relação ao protótipo, impactando a percepção de qualidade da UI.
- Pode causar interpretação incorreta do progresso do usuário no fluxo, principalmente em telas com múltiplas etapas.
